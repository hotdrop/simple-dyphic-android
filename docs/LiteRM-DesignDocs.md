# LiteRT-LM 導入設計ドキュメント

## 1. LiteRT-LMとは
`LiteRT-LM`はオンデバイスで動作するLLMを「Chat APIとして扱うためのSDK」である
Androidアプリ内からLLM（Gemma系モデル）をAPI的に呼び出す抽象を実装することで高品質なローカルLLMアクセスを実現できると考える。
ここでいう「API的」とは以下を意味する: 

- リクエスト → レスポンスのインターフェースを持つ
- ストリーミング応答が可能
- 会話コンテキストを保持できる
- 実装詳細（ローカル）を隠蔽できる

## 2. アーキテクチャ方針の際のアドバイス
LLMを「外部API」ではなく「アプリ内部コンポーネント」として扱う方針が良い。従ってLLM呼び出しは **インターフェースで抽象化** することが望ましい。責務の分離をすることでオフライン対応やプライバシー制御、コスト最適化が可能になる。
LLM応答はストリーミング前提で設計するのが良いと考える。完全レスポンス待ちはUXが悪いことや`LiteRT-LM`は逐次生成が可能という点が理由である。推奨方法として、FlowをそのままUIに流すかtyping表現を入れる。

## 3. LiteRT-LM の利用モデル
- `LiteRT-LM`のEngine:
  - モデルのロードと実行環境の管理
  - アプリ起動時に初期化
- モデル管理:
  - モデルは `.litertlm` 形式で扱う
  - 事前に直接アクセス可能な場所に配置する必要がある。アプリ内ダウンロードではなく、ユーザーがHuggingFaceから取得して配置してもらう。
  - 配置方法: assets or internal storage
