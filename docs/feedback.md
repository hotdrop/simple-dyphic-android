# 2026-04-12 LiteRT-LM / Gemma 4 AIアドバイス機能

## 今回の指示は妥当だったか
- 妥当だった。目的、対象指標、配置したい UI、設定で可変にしたい項目が明確で、実装判断に必要な条件が揃っていた。

## 実装計画書を作成するにあたってのフィードバック
- `LiteRM-ReadMe.md` と `LiteRM-DesignDocs.md` の参照指示があったため、LiteRT-LM を「外部 API ではなくアプリ内部コンポーネント」として扱う判断がしやすかった。
- 一方で依存 version はドキュメント記述と実配布版のズレがあり、`latest.release` を採る前提を先に書いてもらえると往復が減る。

## 改善してほしい指示
- `Gemma 4` の想定モデルサイズや、最低限ターゲットにしたい端末性能があると、GPU 前提の UX や初回ロード案内をさらに詰めやすい。
- 手動生成のみでよいかは今回明示されていたが、将来の自動生成予定があるかも併記されると ViewModel 設計の先回りがしやすい。

## 既存ルールの改善
- `AGENTS.md` の「ActPlan は Plan 専用・CompleteReport/feedback は追記のみ」というルールは今回かなり有効だった。
- 追加候補として、外部 SDK 導入時に「公式 docs に `latest.release` 指定がある場合の version catalog 運用方針」を一文持つと依存追加判断が早くなる。

## ルール更新要否
- 任意の改善候補はあるが、現時点で必須更新までは不要。
- もし更新するなら、`latest.release` を採るケースと固定 version に pin するケースの運用基準を `AGENTS.md` か `skills/` に追記するとよい。
